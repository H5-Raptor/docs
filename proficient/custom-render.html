一般性情况下，业务方将 raptor 提供的预览 html 文件自行存档且对外提供业务域名的 url 供访问即可，如果业务侧需要介入预览页逻辑， 可以继承预览页模块二次开发，在关键的钩子方法中遍历页面数据做修改，或者直接在 html 页面插入一段 script 脚本也是可以的。

特殊的，iframe 预览页并非要求以组件为基础单位的方案，拿到上述页面的数据，iframe 预览页完全可以实现以素材库为基础单位的方案。 后续再述~
